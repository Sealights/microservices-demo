Sealights Boutique Project:
Get Started!

Welcome to the Sealights Boutique Project! This project is designed to be your one-stop shop for learning about Sealights and its powerful features.

What You'll Get:
  • Boutique GitHub Repository: Access the codebase for the project, built upon Google's popular microservices-demo project (https://github.com/microservices-demo) We've integrated Sealights for monitoring and testing, along with customizations and automated testing pipelines using Jenkins.
    • Pre-Configured EC2 Instance:
      • Boutique Running (Port 8081):  Explore a ready-to-go instance showcasing the boutique project in action.
      • Jenkins with Prepared Jobs (Port 8080): Easily rerun key workflows and see how Sealights integrates with your CI.
  • Sealights Account (if needed): We'll provide a Sealights account if you don't already have one.
  • Comprehensive Documentation:  Dive deep with our detailed documentation ( https://docs.sealights.io/boutique-project ), covering everything from microservices and tests to achieving code coverage and configuration using YAML.
Getting Started Tips:
  • Read the Documentation: We recommend starting with the documentation (or referencing it as you explore) for a comprehensive understanding of the project.
  • Need Help? Contact Customer Success:  Our friendly customer success team is here to answer your questions. They're happy to help, and if it's a common question, we might even add it to the documentation for future users!
  • Experimentation Encouraged: This project is your playground! Feel free to experiment without worrying about breaking your CI. If something goes wrong, our customer success team can help you fix it, or we can simply create a fresh EC2 instance for you.
  • Single-User EC2: This EC2 instance is designed to run one CI process at a time. Running multiple processes simultaneously can slow things down and potentially cause errors.
  • Open for Improvement:  We value your feedback! If you have ideas, notes, or complaints, or want to contribute to the project, reach out to customer support or open an issue/pull request on the main repository ( https://github.com/Sealights/microservices-demo-template).
